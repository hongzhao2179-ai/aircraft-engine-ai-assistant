# 产品需求文档 PRD
## 航空发动机智能故障维保助手

## 1. 项目背景
在航空发动机维修过程中，工程师需要查阅大量维修手册、故障案例和技术文档来判断故障原因和维修方案。传统方式依赖人工查阅文档和经验，效率较低。

本项目旨在构建一个基于大语言模型 + 知识图谱的智能故障维保助手，实现故障问答、故障推理、维修知识查询等功能。

---

## 2. MVP产品目标
MVP版本需要实现以下核心能力：

1. 用户可以输入故障问题，AI 给出维修建议
2. 用户可以输入故障症状，AI 推理可能故障原因
3. 系统可以从 Neo4j 知识图谱中查询相关故障知识
4. 支持多轮对话
5. 对话记录保存在本地
6. AI回答支持 Markdown 格式
7. AI回答流式输出（逐字显示）

---

## 3. 核心功能

### 3.1 故障问答
用户输入问题，例如：

发动机温度过高怎么办？

系统流程：
用户问题 → Neo4j 查询 → LLM 生成答案 → 返回结果

AI返回内容包括：
- 可能故障原因
- 检查步骤
- 维修建议

---

### 3.2 故障推理
用户输入症状，例如：

症状：振动大、温度高、噪音

AI输出：
- 可能故障
- 建议检查项目
- 维修建议

---

### 3.3 知识图谱查询
用户可以查询设备、部件、故障、维修方法之间的关系。

例如：
查询：燃油泵

返回：
- 燃油泵 → 故障 → 压力不足
- 燃油泵 → 维修 → 更换燃油泵
- 燃油泵 → 属于 → 燃油系统

---

### 3.4 多轮对话
系统需要记住上下文，例如：

用户：燃油泵压力不足怎么办？
AI：建议更换燃油泵
用户：需要检查什么？
AI：需要检查燃油滤清器和燃油管路

---

### 3.5 对话记录保存
要求：
- 对话记录保存在 LocalStorage
- 刷新页面不会丢失
- 可以清空对话

---

## 4. 界面要求
界面类似聊天界面：

- 用户消息在右边
- AI消息在左边
- 底部输入框
- 发送按钮
- 支持 Markdown
- 支持代码高亮
- 支持流式输出
- 简洁现代风格
